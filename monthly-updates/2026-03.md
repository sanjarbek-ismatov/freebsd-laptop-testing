# Laptop Integration Testing Update - March 2026

## Summary

In March, the Laptop Integration Testing project primarily investigated holistic KDE user experience issues and improved setup instructions/documentation for GPU and Wi-Fi device passthrough. Along the way, we continued iterating on our Contributing Guidelines and automation to make community-driven testing as easy as possible.

## Completed

* Ly is now at the latest upstream version in ports tree: [D55647 - x11/ly: update to 1.3.2](https://reviews.freebsd.org/D55647)
* Lots of FreeBSD-specific pipewire fixes have been upstreamed. Pipewire and wireplumber are now at their latest upstream versions in the ports tree: [D55821 - multimedia/pipewire: update to 1.6.2](https://reviews.freebsd.org/D55821)
* All issues with testing laptops from live FreeBSD installations are resolved. Plain mfsBSD is now the recommended FreeBSD image for community-driven Laptop Integration Testing: [Getting Started with Laptop Testing](https://github.com/FreeBSDFoundation/freebsd-laptop-testing/blob/main/CONTRIBUTING.md#getting-started)
* The first release of the [FreeBSD Laptop Compatibility Matrix](https://freebsdfoundation.github.io/freebsd-laptop-testing/) website is now up. Please feel free to provide your feedback by starting a discussion on our [Github repo](https://github.com/FreeBSDFoundation/freebsd-laptop-testing/discussions).

## In Progress

* We have procured the necessary hardware to refresh the instructions on how to do PCI device passthrough (mainly GPU and network devices) with bhyve, targeting Windows VMs to start. We expect the updated documentation to be merged into vm-bhyve soon.
* We have started doing a revamp of the Wayland compositor setup instructions in the [FreeBSD Handbook](https://docs.freebsd.org/en/books/handbook).

## Challenges

* Large foundational ports (such as x11/kde) are frequently missing from pkg.freebsd.org, because our Poudriere port builders never end up building them. These ports get queued in Poudriere, but their host hardware goes idle before the queue completes. For example, see https://pkg-status.freebsd.org/beefy23/build.html?mastername=150amd64-default&build=c941e5b33dea. In this case, x11/kde gets queued, but the system goes idle before completing the build, resulting in a missing binary pkg for that arch: https://www.freshports.org/x11/kde.

## Next Steps

* Migrate Laptop Compatibility Matrix website to freebsd-doc static site generator. This will take advantage of FreeBSD CSS styling and theming.
* Send out a community call-for-testing to get initial testing contributions
* Submit end-to-end test reports for all laptops available to the FreeBSD Foundation's developers
